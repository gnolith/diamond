# Support policy

Until version 1.0.0, only the latest pre-release revision is supported and
breaking changes may occur. After 1.0.0, the project intends to support the
current major release and provide security fixes for the immediately previous
major release for at least six months after supersession.

Use GitHub Issues for reproducible bugs, feature requests, documentation
problems, and usage questions. Security reports follow `SECURITY.md` and must
not be filed publicly. The sole maintainer provides best-effort community
support with no response-time or service-level guarantee.
